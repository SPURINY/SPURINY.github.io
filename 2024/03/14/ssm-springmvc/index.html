二，springmvc接收数据

2.1 访问路径设置

@RequestMapping( )把handler(方法)和访问路径注册到HandlerMapping中

  • @WebServlet("**/**xxxx") 路径要求严格，必须以“/ ”开头
  • @RequestMapping("xxx")要求没那么严格
    • 支持模糊匹配：/user/*——->/user/a或/user/b都可以匹配上
    • /user/**———>/user/a/b/c甚至都能匹配上
  • 类上加@RequestMapping("xxx")时能对类内所有方法的路径加一个前缀，此时方法上的成了进一步的具体路径
    • 若类上写了，方法上可以只写@RequestMapping，这样访问该方法路径就是类上的值
  • 请求方式，若不满足指定的请求方式—-报405
    • 或用@PostMapping，@GetMapping等代替@RequestMapping

2.2 接收参数

1.param与JSON参数对比

param：k1=v1&kk2=v2

  • GET

JSON: k1:v1,k2:v2【可用于复杂数据结构]

  • POST

handler方法形参处默认什么注解都不加时是param数据，json加@RequestBody注解，路径参数加@PathVariable注解

除param数据当形参外，其他类型数据当形参都需要加注解@RequestBody

2.param参数接收

① 直接接收 /param/data?name=xxx&age=xxx

  • 可以不传递，或传错参数名，——-不报错

访问http://localhost:8080/param/data?namesssss=root&age=19【参数名和形参列表对不上】

显示：name:null,age:19

② @RequestParam注解(加在形参列表上)

  • 可设定不必须传某参数，并设默认值-------------@RequestParam(required = false,defaultValue = “xx”)
  • 可设定必须传，不穿报错400---------@RequestParam

③一名对应多个值(用集合接收+@RequestParam) hbs=xx1&hbs=xx2

访问：http://localhost:8080/param/data03?hbs=%22sleep%22&hbs=%22music%22

p.s.:param底层是asc编码，所以””复制过来变了

④ 用实体类接收

直接在形参处写上实体类

访问：http://localhost:8080/param/data04?name=root&age=19

显示：User{name='root', age=19}

3.json数据接收

json数据和实体类属性对应，用实体类接收JSON数据

用postman可以设置请求体，

结果：>HTTP状态 415 - 不支持的媒体类型

  • 因为java原生的api只支持param参数和路径参数，json属于前端格式
  • solution：
    • ①导入json处理依赖[jackson-databind]
    • ②给HandleiAdapter配置json转化器 [在配置类上加@EnableWebMvc注解]

结果：

@EnableWebMvc相当于添加一个HandlerAdapter和HandlerMapping，还给HandleiAdapter配置json转化器

  • 所以配置类加了@EnableWebMvc注解后可以不写HandlerAdapter和HandlerMapping的bean

  • 重定向：（二次请求）可以是项目外的

    • 用springmvc重定向项目外地址belike：return "redirect:https://www.bilibili.com"

3.返回JSON数据

  • 接收json数据：
    • 在形参上加@RequestBody注解
    • 形参是实体类
  • 响应json数据
    • 返回类型为实体类
    • 方法上加@ResponseBody
    • HandlerAdapter把返回的实体java对象转化成 json字符，又因为有@ResponseBody注解，所以不会去找视图解释器，直接返回json字符串

访问路径：http://localhost:8080/json/data , 结果：{"name":"purin","age":20}(成功响应成json数据)

http://localhost:8080/json/data02 —–>[{"name":"purin02","age":21}]

补充

@RestController=@Controller+@ResponseBody

@ResponseBody：数据直接放到响应体带回，不走视图解析器查找视图(返回jsp)，也不能转发和重定向

4.返回静态资源

  • 静态资源不能放在WEB-INF下（外部访问不了

  • 发现部署项目后无法通过 …8080/imgs/img1.png访问，因为↓

    • DispatcherServlet先去秘书HandlerMapping找是否有对应路径的handler（方法），然后才访问，而图片等静态资源没有handler

  • solution：配置类里写xxx，开启静态资源查找

    • @Override
//开启静态资源查找
//DispatcherServlet--->HandlerMapping去找有没有对应的handler--》没有--》查找静态资源
public void configureDefaultServletHandling(DefaultServletHandlerConfigurer configurer){
    configurer.enable();
}

五，声明式异常处理

  • 原来：在业务代码里写异常处理

  • 现：把异常处理的代码提取出来

  • 发生异常时交由专门的异常处理Handler

  • //发生异常时，会走此类的handler，一个一个匹配，找到了(异常类型匹配)就over
@RestControllerAdvice//=ResponseBody+ControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler
    public Object ArithmeticExceptionHandler(ArithmeticException e){
        String s=e.getMessage();
        System.out.println("exception message:"+s);
        return s;
    }
    @ExceptionHandler
    public Object ExceptionHandler(Exception e){
        String s=e.getMessage();
        System.out.println("exception message:"+s);
        return s;
    }
}
